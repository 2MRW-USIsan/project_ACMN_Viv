# 画面実装依頼書テンプレート

このドキュメントは **AI への画面実装依頼時に記述する入力ドキュメント** のフォーマットです。  
「要件定義書」または「実装指示書」として、何を・どのような目的で作るかを簡潔に伝えることを目的とします。  
実装の詳細（コンポーネント分割・型設計・フック構成など）はAI側で判断するため、このドキュメントでは記述不要です。

テンプレートの記入例として [`/editor` 画面の依頼書](#editor-画面実装依頼書) を末尾に添付しています。

> AI が実装完了後に出力する成果物報告書は [`SCREEN_REPORT_TEMPLATE.md`](./SCREEN_REPORT_TEMPLATE.md) を参照してください。

## `/editor` 画面実装依頼書

---

### 1. 画面基本情報

| 項目 | 内容 |
|---|---|
| 画面パス | `/editor` |
| 画面名称 | Bloc リストエディタ |
| 目的・概要 | AI プロンプト用の「Bloc」設定データを GUI で組み立て、YAML 形式でエクスポートする画面。Bloc は Orders・Select・Switch という3種類のドメインを持つ設定単位で、複数の Bloc をリスト形式で管理できるようにしたい。 |

---

### 2. 実装してほしい機能

- Bloc をリスト形式で表示・追加・削除できる
- 各 Bloc は key・label を入力でき、折りたたんで表示を省略できる
- 各 Bloc に対して Orders・Select・Switch のいずれかのドメインを有効化できる（チップで切り替え）
- 有効化したドメインに対してアイテムを追加・編集・削除できる
- 編集内容を YAML 形式でプレビュー・クリップボードコピー・ファイルダウンロードできる
- 編集内容をサーバー（API）に保存でき、保存済みデータをロードして復元できる
- ロード済みデータと現在の入力内容の差分を検知して警告を出せる

---

### 3. UI 要件

**レイアウトイメージ：**

```
- 上部バー：「Generate YAML」ボタン、保存済みデータの選択ドロップダウン、ロードボタン
- 本体：Bloc のリスト（各 Bloc は折りたたみパネル）
  - パネル内：key・label の入力フィールド、ドメイン切替チップ、選択ドメインのアイテムリスト
```

**主要な UI 要素：**

- Generate YAML ボタン：YAML プレビューダイアログを開く
- 保存済みデータ選択ドロップダウン：保存済みスナップショットの一覧から選択
- ロード/再選択ボタン：選択したスナップショットを反映 / 選択状態をリセット
- Bloc リストの追加ボタン：新しい Bloc パネルを末尾に追加
- ドメインチップ（Orders / Select / Switch）：クリックで該当ドメインの有効/無効を切り替え
- YAML プレビューダイアログ：YAML テキスト表示、コピー・ダウンロード・DB 登録ボタン

---

### 4. データ・API 要件

**扱うデータ：**

- Bloc リスト（key・label・ドメイン設定を持つ構造体のリスト）
  - Orders ドメイン：アイテム → 子フィールド → オプション値のネスト構造
  - Select ドメイン：アイテム → 子フィールド → 選択肢リストのネスト構造
  - Switch ドメイン：アイテム → on/off の2値フィールドのネスト構造
- 保存済みスナップショット（名前・作成日時・Bloc リストの状態を持つ）

**必要な API 操作：**

- 保存済みスナップショット一覧の取得
- ID を指定してスナップショット詳細の取得（ロード）
- 現在の Bloc リスト状態を新規スナップショットとして保存

---

### 5. 技術的制約・参照すべき規約

- `ARCHITECTURE.md` / `GUIDELINES.md` に従った MVVM + Atomic Design + Clean Architecture 構成で実装すること
- 状態管理は `useReducer` を基本とし、ドメインごとにリデューサーを分割すること
- コンポーネントは描画のみ、ロジックはカスタムフックに分離すること
- YAML 出力は `yaml` パッケージを使用し、内部 ID や状態フラグは出力に含めないこと
- API の実装は当面インメモリモックで可（将来 DB に差し替えられる構造にしてほしい）

---

### 6. 補足・未確定事項

- Orders の子フィールドには複数の type（Random・Scripts・Color・Complex 等）があり、type ごとに入力 UI が異なる。詳細は実装時に都度確認するか、適切な設計でAIに判断してほしい
- Select の選択肢にはシャッフル機能、Switch にはランダマイズ機能が必要
- 保存名は自動生成（例：「保存 2026/01/01 12:00:00」）で可
